平台Launcher配置流程

平台Launcher配置流程

烧机

使用：

  1. 使用北都aml_upgrade_package_2019-12-18的镜像进行烧录
  2. 盒子需要正式的数据 不能随便搞一个mac sn deviceId ，烧录以后修改，需要跟陈彬彬拿
  3. 证书会用到Build.MODEL ，所以用什么镜像也会影响到证书，北都镜像MODEL是LINKTV01

烧录SN 等信息

串口或者adb输入命令，设置SN

串口或者adb输入命令，设置deviceID

重启检查是否已经将数据写入到节点中

替换两个APK，何龙提供的

替换SkyCertStorageService.apk 原来的在/system/app/SkyCertStorageService下面
替换后清楚/data/data/com.skyworthdigital.stb.cloud.certmanagerserver的数据

替换SkyworthProtocolServer.apk 在/system/priv-app/SkyProtocolServer下面
替换后清楚/data/data/com.skyworthdigital.stb.cloud.connect.server的数据

如果无法读写，请手动添加权限白名单etc/permissions/privapp-permissions-atv.xml

修改数据生成新的证书

SkyCertRequest_new\SkyCertRequest\skyworthResource\inputData
中有两个文件：config.xml 和 dataList.txt

config.xml,例子中用的是北都的配置，平台也有设置

dataList.txt,内部数据是 MAC 空格 DeviceID

配置后开始生成证书

双击SkyCertRequest下的SkyCertRequest.exe生成证书

证书生成在output目录下
CMC开头的是公共的证书，所有盒子都一样
剩下的两个都是盒子的证书，每个盒子都不一样

使用WritCert 将证书写入盒子

  1. 把你刚才生成的证书文件直接放到WritCert代码的asset目录下（全部放入）apk根据盒子信息进行读写。
  2. 这个代码 用平台的签名
  3. 不能将代码放到有中文目录的地方编译运行，否则报错
  4. 这个WritCert需要用到permission com.skyworthdigital.stb.cloud.certmanagerserver.permission.CONNECT_CERT，要提供系统签名，放到/system/app或者/system/priv-app下。
  5. 添加白名单

  1. 启动WritCert

  2. 直接点击前面四个写证书的按钮，返回都是成功

  3. 点击READCMCCERT 能够看到证书

检查Launcher拿到数据

流程：

  1. 能不能读到预置证书
  2. 拿到设备证书
  3. 拿到数据
    如果出现错误：在logcat打印中搜索“CertManager”这个tag
