ATV Netflix集成教程

二、前置状态：

  • 客户在Neflix approved 立项成功，能够在PDM（Partner Device Manager）查询到项目信息。
  • 在测试部申请，widevine key（需要跑CTS，在首个测试项成功后立刻停止，然后将测试结果，上传Google，可以获取Widevine Key System ID），Playready key ，这些证书与ATV测试相关，与Netflix本身无关，但是Netflix播放本身需要使用这些证书。
  • 产品经理在Netflix approved 后需要进入PDM将产品信息补全。如果是零售业务，一般需要申请一个MGKID，在盒子生产过程中写入。一般来讲我们OEM厂商生产不需要MGKID。

    To avoid delays in the certification process, the following tasks should be completed and approved prior to submitting a device firmware:
    • NRD Features
    • Device Models
    • Android CTS/GTS3

3.2 InterAction ID集成

平台已经做好，如果出现问题：

  1. 待机启动出现问题，需要检查红外遥控器是否支持Netflix的真待机启动

  2. 快捷键启动IID出错（IID-001-* fail**）确认遥控器没有问题以后，则需要Netflix帮忙修改

3.3 调试

准备盒子：

  1. 申请Widevine key，playready key，HDCP ( 2.0 / 4.2) key, Android Q需要provision2.0 key

  2. 一般来讲我们OEM厂商生产不需要MGKID。(Model Group Key ID)

错误分析：

  1. Error(13) ,说明Model name 等fingerprint 与Netflix PDM中的信息不一致，或者key本身也不一致，或者model group 不正确（在Netflix 网页DPM可以查到 model group）

  2. 注意检查有没有Netflix ESN号：Android Q Settings -> Device Preferences -> Advance options -> Netflix ESN

    ESN的组成：前面是Netflix esn的前缀（项目PDM决定）+ Model + Widevine system id + 随机数（类似sn，每个盒子是固定的）
    ESN的长度范围：The total length of a Netflix ESN is from 11 to 42 characters.
    ESN允许的字符：ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789-=
    更多ESN信息：https://docs.nrd.netflix.com/docs/nrdp/cert/partner-device-manager/esn-specification

    实例：
    NFANDROID2-PRV-SKYWORTH-SKYWOOTT-01-17624-AC6D3A55A55A88E0DC095474376D331ABF5ED2398BA111DC94EF1A82A4202665383

    NFANDROID2-PRV-SKYWORTHOTT(PDM 前缀 基础上去掉了OTT)+OTT-01(model)+widevine system id(17624)+DIVICE ID(AC6D3A55A55A88E0DC095474376D331ABF5ED2398BA111DC94EF1A82A4202665383)

    广播获取netFlix ESN的方法：

  1. 检查Widevine key： ESN里面的17624 就是Widevine key system id

  2. DPI Model name dismatch
    检查PDM上DPI model name是否一致，是否多出空格等不容易检查的问题。

  3. Ninja版本在PDM设置错误导致该问题：
    android Q使用Ninja 7 就可以了。Android R才会用到Ninjia8.PDM上会有Ninja版本的设置。

  4. Netflix按键无法真待机唤醒，需要在这个文件替换红外用户码

四、测试

在研发阶段就要先预约测试资源：当前测试部3人负责Netflix 测试，Netflix测试的硬件设备和网络环境是有限的。
需要预约测试人力以及设备资源。另外，需要提前知会Amlogic等方案商，帮忙测试，并且需要准备2台盒子给到方案商帮忙测试和处理问题。

测试内容有四个部分：

  1. Auto 自动测试，直接跑就可以，测试项中有AT的标志
  2. Manual 手动测试，需要手动点击。
  3. EP 需要设备，测试间可以进行。
  4. HS 需要设备，Amlogic帮忙测试。

测试环境：

  1. 需要Netflix RAE设备（测试部），从设备出来的网络是Netflix测试网络

  1. 连接Netflix测试网络以后，打开Netflix如下

  1. 另外应该在NTS网页进行测试 https://partnertools.nrd.netflix.com/nts/#testplan

提交测试：

下载DirectvGO项目Netflix认证测试申请表&Checklist&自测试报告(V1.1) .xls，并填写表格，将里面的自测项目测试通过。
\\192.168.3.11\海外运营测试专用\文件模板\测试申请表\Netflix认证测试

  1. 需要提供user版本以及eng版本。
  2. 需要启用secure boot。
  3. 需要准备测试的6个盒子+6个遥控器

注意事项

  1. 遥控器Neflix 按键唤醒，需要遥控器增加特殊的红外脉冲，开机唤醒。

  2. 下载最新Netflix apk内置：检查链接https://docs.nrd.netflix.com/docs/nrdp/dev/android-tv/android-tv-downloads

  3. 遇到model group name的空格全部去掉后编译。例如：阿根廷项目

  4. finger print 中Brand： Brand不能有空格，不能有下划线A-Z, a-z, 0-9, -。因为会被Netflix用到，按照Netflix的ModelGroup的格式要求，不能出现下划线“_”，如果“-”也用不了的话，那Brand就只能是数字和字母了。
    不过我看到之前有项目的Brand是用了“-”的，比如Strong项目的Brand是“STRONG-ZWL”，马电项目是“unifi-TV”，iZZi项目是“izzi-telecom”，貌似也没报什么问题

  5. finger print 中Product： Product不能有空格。因为不会被Netflix用到，所以按照我截图的Google CDD文档里面的描述，“^[a-zA-Z0-9_-]+$” 这些应该都是可以的，没有其他限制。我们自己定的规则来说，是“硬件版本-客户名称”，如果客户名称里面包含其他字符，则需要使用Google CDD文档中规定的字符来替换，但应避免重复使用“-”。

  6. 遥控器音量键无法调节音量某些节目：说明缺少MS12 ，不支持杜比Decoder，需要支持杜比 MS12。

  7. 关于IID出错的问题: 001出错的项目需要Netflix帮忙修改PDM的项目信息，而002的fail需要修改具体入口的IID。
    IID-001-* fail： Expected iid of 0b2a9564, but found iid of 0b2e9566
    IID-002-* fail： Expected iid of 4925312e, but found iid of 17ae68f5

    Netflix 的回复：简单来讲就是Netflix需要Skyworth确认 Feature , UI specs 和 remote control ,然后Netflix 才会review并通过，在Netflix review之后才开始测试。

    When we approve a project, we would need Skyworth team to confirm all specs including features, UI specs and remote control information before we actually start integration. Running IIDs in NTS seems too early at this stage when the PDM specs have not been reivewed and approved by Netflix partner engineering and PEMs.

    Regarding IID, for each operator projects, we could define certain type of launch IID differently from ODM reference and Some of the IID is defined by Ninja. This need to reviewed and approved by PEMs.

    For Android specific fix IID: for example, IID-001 Netflix button, if you sent a correct intend then the NInja apk it will return iid of 0b2e9566, this seems an issue in our side I will follow up internally, please skip this

    For IID-002, current UI specs defines as operator project specific IID, fb79567c . from your error message, it Reporting of Launch Event - Netflix icon the IID expected is wrongly matched icon in the app list.

    I would recommend to wait until we finalized the UI specs before you run more tests.

  8. 补丁问题: testplan 中有casePROF-013-TC10/15 ，根据零售项目的信息，需要合入补丁才能在TCLroku 电视上pass（TCL roku只有aml office 有）

  9. 杜比Astom：杜比全景音，如果支持MS12则可以支持透传（意味着盒子本身不解码Astom，但可以传给电视机进行播放，如果电视机支持则播放，不支持则不能播放Astom，相当于没有Astom功能）

  10. DPI-013-TC1运行fail提示：nrdp.devices.SDK Version.ui.width is not equal to 1920.The actual value is:1280
    在盒端打开Netflix以后，使用命令查看信息：dumpsys SurfaceFlinger

    这个问题的解决方法：在PDM上修改UI信息为1080，另外需要Netflix帮忙在后台配置Netflix apk 1080 ui可用。不需要修改源代码。
    在PDM上提的Help tracker与Netflix 沟通：

专有名词：

TEE (Android): trusted execution environment

PDM(Netflix) : Partner Device Manager

CTS(Android) : Compatibility Test Suite

MGK ID(Netflix): Model Group Key ID. The MGK IDis one part of a data tuple which is used to identify and secure an Android device. In addition to the MGK ID, each device also uses a set of security keys. The MGK ID is public, while the security keys MUST be kept secret.

IID(Netflix): Interaction IDs (IIDs) replace Launch Source Types. For more information about IIDs, see the NRDP documentation around App Startup.

ESN(Netflix):Electronic Serial Number. Uniquely identifies a device and starts with an ESN prefix that is generated for you by the Partner Device Manager.

CDM ID(PlayReady):CDM ID (Content Decryption Module ID) is a DRM client identifier. For additional detail, see the CDM ID.

RAE(Netflix): The Netflix Reference Automation Environment (RAE) integrates with the Netflix testing infrastructure, Netflix Test Studio (NTS), to test your device. https://docs.nrd.netflix.com/docs/nrdp/cert/netflix-test-studio/setting-up-nts/

Dolby Atmos/X-DMAX(Dolby)：杜比全景声由杜比实验室研发，于2012年4月24日发布的全新影院音频平台。它突破了传统意义上5.1、7.1声道的概念，能够结合影片内容，呈现出动态的声音效果

Widevine security level

Your device must satisfy Level 1 Security Level requirements, which includes:

  • The following code must run in a hardware-based trusted execution environment (TEE):
    • content decryption
    • decoding
    • rendering
    • license policy enforcement
  • Each device is factory-provisioned with a device-unique Widevine keybox
  • The Widevine system ID must be unique per operator project for all new integrations.
