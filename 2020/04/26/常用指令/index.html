常用指令

常用指令

Linux

  • cd -:返回上次目录
  • mv out/ out_user：将out目录移动到out_user下，没有该目录可自动创建,并且会删除out目录
  • tar zxvf 文件名.tgz -C ./：解压到当前目录
  • tar zxvf 文件名.tgz -C ./android-p：解压到指定目录
  • df -hl:查看磁盘空间信息
  • rm -rf XXX :删除整个文件夹
  • ls -al :查看文件目录详细信息，且有权限信息
  • ls -d fileName:查看单个文件信息，后面可再接参数
  • chmod 777 fileName：更新文件权限为最高：linux权限详解
  • chown system:system wake_unlock:更改用户组，将后面文件改到system,system
  • chmod a+x fileName:设置当前文件具备可执行文件权限
  • touch traces.txt :创建文件
  • cat traces.txt:查看文件
  • cat list:在指定目录下查看当前存在的所有节点
  • echo -n “” > camera.ini：将文件内容清空
  • grep -rnw “xxxx”：全局搜索携带xxx的具体文件：注意是完全匹配，区分大小写，全字匹配，必须一模一样
  • grep -rn “wifiMesh” device/amlogic：在该目录下寻找带wifimesh字样的内容，非完全匹配，只要包含就行
  • grep -rn “unifykey” –include “*.te”：指定文件类型进行查找
  • pm list package -f | grep wifimesh:包含wifimesh的已安装应用包
  • find . -name “wifimesh“:在当前目录下查找包含wifimesh的内容
  • find /data/app -name “wifimesh“: 在指定目录下查找
  • dumpsys activity | grep -i run:查看当前机器正在运行的activity
  • dumpsys window windows | grep mCurrentFocus ：查看最上一层Activity,只显示最上层窗口信息
  • dumpsys power | grep -i wake：查看当前持有带wake的power锁一共有多少
  • settings list global | grep “OTG”：查看settings数据库
  • settings list secure | grep “saver”：查看settings的secure数据库
  • settings put secure sleep_timeout 100000:更改settings secure数据库
  • service list | grep hello：查看后台service数量，只会列出系统service
  • ln -sf 源文件 目标文件 ：更改软链接的实际链接文件，源文件为实际文件，目标文件为软链接文件：
  • ln -sf TVController_V2.3.6.4.apk Nes_Ir_Control.apk：将后者指向前面那个具体的文件
  • logcat -b main -b system -b kernel ：打印kernel日志
  • pm list -f | grep “com.xxx.xxx” 查看app的安装apk位置信息

烧录SN 等信息

读取系统属性，及系统属性的意义：

  • persist.sys.audio_output.mode=2：前面带persist的属性，代表只是首次进入系统的时候读取的预制值
  • ro.use.nes.skyway:这种前面以ro开头的属性是没法在代码里更改的
  • sys.nes.allow.global.keyevent:这种值是在本次有效，但是开完机之后就会恢复最原先的值
  • adb shell setprop persist.sys.nes.print.log 1：设置机器日志打印的属性
  • getprop:列出所有的系统属性
  • adb shell getprop persist.sys.nes.print.log
  • adb shell getprop | grep print.log

读取属性的正确方式是这样，因为看build.prop有些参数例如persist开头的表示的可能不是当前的值：

  • cat /storage/emulated/0/.priv > write:将文件中的内容直接写进去
  • > echo keyValue > write ：将string内容写进去

  • rm /data/smarthome/iot_zigbee.*:删除该目录下iot_zigbee.开头所有的内容。
  • ps ：查看当前多少进程
  • ps | grep device :查看带device名的进程(注意有些盒子需要加上busybox关键字)eg:busybox ps | grep device
  • busybox ps ：在盒子上通过这个命令看进程。ps看不到！！！ 同理，busybox ps | grep device_manage 可以查看指定名字进程信息
  • pm clear + 包名：清楚该包名的数据
  • pkill device_manage
    kill -9 进程号
  • mv device_managee device_manage:将前者命令为后者
