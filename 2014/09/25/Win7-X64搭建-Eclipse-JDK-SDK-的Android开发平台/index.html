Win7 X64搭建 Eclipse+JDK+SDK 的Android开发平台

Eclipse+JDK+SDK 的Android开发平台

一 安装配置Eclipse环境

1 下载安装包

ADT:adt-bundle-windows-x86_64-20140702.zip

下载地址：https://dl.google.com/android/adt/adt-bundle-windows-x86_64-20140702.zip

Java：jdk-8u20-windows-x64.exe

下载地址：http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

2 安装下载包

Java： 点击安装包，下一步、下一步即可。

这个时候还要通过一系列的环境变量的配置才能使用JDK环境进行JAVA开发。配置环境变量包括java_home，path和classpath三个部分(大小写不敏感)。

用鼠标右击“我的电脑”，选择“属性”->“高级”->“环境变量”->“系统变量”->“新建”。

在“变量名”输入框中写入"java_home"，在“变量值”输入框中写入
D:\Program Files\Java\jdk1.8.0_20 （根据安装路径填写），然后点击“确定”，java_home就设置完成了。

下面开始”classpath”的配置。选中“系统变量”查看是否有classpath项目，如果没有就点击“新建”，如果已经存在就选中classpath选项，点击“编辑”按钮，然后在“变量名”中填写”classpath”，在“变量值”中填写
.;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar

现在可以进行“path”的配置了。同上在“classpath”设定时类似，“变量名”输入框填写“path”，“变量值”输入框填写
%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin

JDK的环境变量已经配置完成，可以通过打开命令提示符窗口，输入命令java -version， 看到Java版本的信息，来确定安装是否成功。首先点击“开始”，然后点击“运行”(或者win+R)。
然后会出现下图所示的输入界面，输入“cmd”,按”Enter”键。

这个时候就进入了“命令提示符”窗口。这个时候输入
java -version,特别注意 java和-version之间有一个空格，按”Enter”键。

javac -version，按”Enter”键。
JDK版本信息就全部显示出来了，也表示JDK已经安装和配置完成。

Eclipse+SDK:解压adt-bundle-windows-x86_64-20140702.zip即可。
这里我解压到了D：

配置SDK环境变量：在“变量名”输入框中写入“android_home”，在“变量值”输入框中写入D:\adt-bundle-windows-x86_64-20140702\sdk （根据安装路径填写），然后点击“确定”，android_home就设置完成了。

现在可以进行“path”的配置了。同上在“classpath”设定时类似，“变量名”输入框填写“path”，“变量值”输入框填写
%android_home%\tools;%android_home%\platform-tools;

这个时候就进入了“命令提示符”窗口。这个时候输入”adb”和“draw9patch”

可以看到adb的version也可以启动draw9patch的时候就设置成功。

打开SDK Manager.

这里可以下载查看已经安装的SDK。这里安装了Android 4.4W。
选择需要的SDK下载。这里就不演示了。

打开Eclipse就可以开始Android之旅了。

设置工作目录。

新建Android工程。File->New->Android Application Project.

填写项目名称，选择最低支持的SDK。其它默认即可。

生成项目后尝试运行。右键点击工程->Run As->Android Application.

还没有设置AVD模拟器，会提示设置，Yes即可。

新建AVD

启动模拟器。

这样的AVD只能运行android 4.4W项目。如果运行新建的这个非Android手表工程会报错。

可以打开SDK Manager 下载不是W结尾的SDK可以解决。
