二、运行缓慢

在Win10中使用bash 命令行会特别慢，我们使用CMD则会特别快。（win7中暂无此问题，暂不知道具体原因）
解决方案： 使用CMD代替bash。

已解决： 当CMD中的git 中文支持不好时，就会出现git 命令特别缓慢的情况，当解决CMD乱码以后，缓慢问题就解决了。

三、 设置大小写敏感

core.ignoreCase 用来控制git是否忽略大小写，
设置为true之后，这个选项能够让git更好的工作在很多无大小写区分的文件系统，如FAT。
设置为false之后，git会区分文件名大小写。
